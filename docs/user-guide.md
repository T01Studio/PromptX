## 1. PromptX 是什么

PromptX（灵犀）是一个**提示词管理引擎**。

你在日常使用 ChatGPT、DeepSeek、Kimi 等 AI 工具时，是否遇到过这些痛点：

- 每次聊天都要从笔记软件里复制粘贴相同的提示词开头
- 提示词里的 `{{变量}}` 要手动查找替换，一不留神就删错
- 同一个任务，不同提示词的效果好坏全靠感觉，没法系统对比

PromptX 的解决思路很简单：**你管理提示词模板，AI 帮你填参数**。

架构上，它是一套工具组合：

| 组件 | 说明 | 对象 |
|------|------|------|
| **Web 应用** | GitHub Pages 托管的前端，核心操作界面 | 所有人 |
| **浏览器扩展** | 安装在 Chrome 中，在任何网页输入框里按 `\` 就能唤起提示词 | 深度用户 |

> **当前文档聚焦 Web 应用。** 浏览器扩展需要从 Chrome Web Store 安装，独立使用。

## 2. 快速上手指南（5 分钟）

### 2.1 打开应用

访问：**[https://t01studio.github.io/PromptX/](https://t01studio.github.io/PromptX/)**

你会看到一个深色主题界面，左侧有四个导航入口：

```
PromptX
─────────────────
🚗 自动驾驶舱      ← 主操作区
📦 资产管理        ← 管理提示词库
⚔️  A/B 竞技场     ← 对比测试
⚙️  API 配置       ← 设置 AI 密钥
```

### 2.2 三步跑通第一个提示词

**第一步：进入「自动驾驶舱」**

点击左侧「自动驾驶舱」，看到对话界面。

**第二步：输入 `/xhs` 然后按 Enter**

在底部输入框输入 `/xhs`，会弹出一个悬浮菜单，显示「小红书种草文案」。按 `Enter` 或点击它。

**第三步：输入意图并发送**

输入框下方会显示已选择的模板名。现在输入你的意图，比如：

```
帮我写一篇杭州小众咖啡馆探店的文案
```

按 `Enter` 发送。系统会自动识别参数（主题、风格、字数）并展示确认面板。

**第四步：确认参数并生成**

检查面板中各参数的值，如需修改点击「微调参数」，确认后点击「直接生成」。

最终提示词会显示在对话区域，鼠标悬停时右上角出现复制按钮，直接复制到 ChatGPT 等工具使用即可。

> **五分钟你学会了核心流程：选模板 → 说意图 → 确认参数 → 得到提示词。**

## 3. 四大模块详解

### 3.1 自动驾驶舱

这是 PromptX 的核心交互区，用一个对话框完成提示词生成。

#### 快捷指令唤起（Slash Command）

| 操作 | 效果 |
|------|------|
| 输入 `/xhs` | 唤起「小红书种草文案」模板 |
| 输入 `/refactor` | 唤起「代码重构专家」模板 |
| 输入 `/report` | 唤起「工作周报生成器」模板 |
| 只输入 `/` 后不跟字母 | 展示全部可用模板供浏览 |

#### 模板选择后

输入框左侧会显示已选模板名称（如「小红书种草文案」）。此时输入一段简单描述即可，不需要写完整的提示词。

#### 参数确认面板

发送意图后，系统会在输入框上方弹出一个面板，展示自动推断的参数。例如：

```
topic: 咖啡探店
tone: 真实体验/情绪饱满
length: 300
```

你可以直接点击「直接生成」接受这些参数，或点击「微调参数」手动修改每个值。

#### 生成结果

最终提示词会显示在对话区域，格式如下：

```
你是一个小红书爆款文案专家。请为主题：咖啡探店 撰写一篇种草文案。
风格要求：真实体验/情绪饱满
大概字数：300字
要求：包含情绪钩子，使用丰富的Emoji，结构清晰，文末带上相关话题。
```

鼠标悬停在消息上，右上角出现📋复制按钮，一键复制用。

#### 快捷键速记

| 按键 | 作用 |
|------|------|
| `/` | 唤起模板菜单 |
| `↑` `↓` | 在模板列表中上下选择 |
| `Enter` / `Tab` | 选中当前高亮模板 |
| `Escape` | 关闭模板菜单 |
| `Enter`（输入框内）| 发送意图 |
| `Backspace`（输入框为空时）| 取消已选模板 |

---

### 3.2 资产管理

这里是你的「提示词图书馆」。管理所有模板的增删改查和版本历史。

#### 界面布局

| 区域 | 功能 |
|------|------|
| 左侧面板（320px） | 提示词列表，支持搜索和分类筛选 |
| 中间主区域 | 模板编辑器（可编辑提示词文本） |
| 右侧面板（320px） | 版本历史时间线 |

#### 浏览和筛选

- **搜索**：顶部搜索框输入关键词（可按名称或命令搜索，如输入「代码」）
- **分类筛选**：点击「全部」「自媒体爆款」「代码重构」「公文写作」「艺术绘画」切换

#### 编辑提示词模板

点击左侧任意提示词后，中间区域展示：

```
底层模板 (Template)
┌──────────────────────────────────────┐
│ 你是一个小红书爆款文案专家。          │
│ 请为主题：{{topic}} 撰写一篇种草文案。│
│ ...                                   │
└──────────────────────────────────────┘
```

模板中的 `{{变量名}}` 会被系统自动识别为可配置参数。修改后点击右上角「保存快照」即可创建新版本。

#### 版本管理

右侧「版本追踪」面板会列出所有历史快照。每条记录显示时间戳和「回滚至此版本」按钮。点击即可恢复。

> **建议：每次重大修改后保存快照，方便回退。**

#### 新增提示词模板（两种方式）

点击左上角 ➕ 按钮：

**方式一：对话提纯（推荐）**

1. 选择「对话提纯」标签
2. 把你和 AI 的历史聊天记录粘贴到文本框
3. 点击「一键提取」
4. 系统自动分析并结构化输出：能力名称、唤起命令、自动变量、底层模板
5. 确认后点击「保存至资产库」

**示例输入**：
```
我：帮我写小红书探店文案
AI：好的，什么主题？
我：杭州的独立咖啡馆
AI：好的，风格上有什么偏好？
我：温暖治愈风
AI：字数呢？
我：300字左右
AI：[生成的文案...]
```

系统会提取出 `{{主题}}` `{{风格}}` `{{字数}}` 三个变量，并生成标准模板。

**方式二：手动创建**

直接填写表单创建模板。（当前版本此功能演示中，推荐使用对话提纯。）

### 3.3 A/B 竞技场

用同一个测试意图，同时运行两个不同的提示词模板，直观对比生成质量。

#### 使用步骤

1. **输入测试意图**：在顶部输入框写同一句话，如 `帮我写一段探店文案`
2. **选择两个对比方案**：
   - 左侧下拉选择方案 A（如「小红书种草文案」）
   - 右侧下拉选择方案 B（如其他模板）
3. **点击「开始对比测试」**：两侧同时流式输出结果
4. **对比结果并决定**：阅读两侧输出，点击「晋级为主力版本」确定获胜方

#### 应用场景

| 场景 | 对比内容 |
|------|----------|
| 提示词迭代 | 旧版 vs 新版，看哪个更符合你的风格 |
| 风格探索 | 「专业严谨」 vs 「网感/种草」，看你的内容适合哪种 |
| 模型对比 | 在 API 配置页切换模型后，用相同提示词对比 Gemini vs DeepSeek |

### 3.4 API 配置

配置你连接 AI 大模型的方式。所有配置仅保存在你的浏览器中，**不会上传到任何服务器**。

#### 支持的 AI 服务商

| 服务商 | 默认接口地址 | 可用模型 |
|--------|-------------|----------|
| **Google Gemini** | `generativelanguage.googleapis.com` | gemini-2.5-flash、gemini-2.5-pro、gemini-2.0-flash |
| **DeepSeek** | `api.deepseek.com` | deepseek-chat、deepseek-reasoner |
| **OpenAI** | `api.openai.com` | gpt-4o、gpt-4o-mini、gpt-4-turbo |
| **自定义** | 随意填入 | 随意填入（兼容 OpenAI 协议） |

#### 配置步骤

1. 点击左侧「API 配置」
2. 选择你要用的服务商
3. 填入 API Key（从对应服务商后台获取）
4. 选择或输入模型名称
5. 点击「保存配置」

#### 如何获取 API Key

| 服务商 | 获取地址 | 说明 |
|--------|---------|------|
| Gemini | [aistudio.google.com/apikey](https://aistudio.google.com/apikey) | 免费额度充足|
| DeepSeek | [platform.deepseek.com/api_keys](https://platform.deepseek.com/api_keys) | 性价比高 |
| OpenAI | [platform.openai.com/api-keys](https://platform.openai.com/api-keys) | 需充值 |

> 自定义接口支持任何兼容 OpenAI `/v1/chat/completions` 协议的服务（如 Ollama 本地部署、One API 聚合网关等）。

#### 安全说明

- API Key 存储在浏览器的 `localStorage` 中
- 每次调用直接从你的浏览器请求对应的 AI 服务商
- PromptX 服务器**不会看到、不会存储、不会中转**你的 Key
- 点击「清除配置」可立即删除

## 4. 完整实战案例

### 案例一：从零创建小红书文案模板

**背景**：你经常需要为不同品牌写小红书种草文案，但每次都要重新构思结构。

**步骤**：

1. **打开 PromptX** → 点击左侧「资产管理」

2. **点击 ➕ 新增** → 选择「对话提纯」

3. **粘贴一段你满意的历史对话**：
   ```
   我：帮我写一篇小红书种草文案
   AI：好的，什么产品？
   我：国货精华液
   AI：需要突出什么卖点？
   我：成分安全、敏感肌可用
   AI：风格偏好？
   我：真实测评风格
   我：字数300左右
   AI：[生成了文案...]
   ```

4. **点击「一键提取」**，系统分析后输出：
   - 能力名称：国货护肤品测评
   - 唤起命令：/skincare
   - 变量：产品、卖点、风格、字数
   - 模板：（结构化提示词）

5. **确认无误后点击「保存至资产库」**

6. **切换到「自动驾驶舱」**，输入 `/skincare`，输入意图 `推荐一款平价眼霜`，系统自动填充参数 → 生成！

> 这就是**从经验到模板**的完整流程。一次提纯，永久复用。

---

### 案例二：打造代码重构专家

**背景**：你经常需要让 AI 帮你 review 代码，或把 Java 转 TypeScript。

**手动创建模板**：

1. 进入「资产管理」→ 选中「代码重构专家」模板（已内置）

2. 如果你想定制，可以直接编辑模板文本：
   ```
   你是一个资深的{{role}}。请分析以下用 {{sourceLang}} 写的代码，指出潜在问题并以 {{targetLang}} 重写。输出格式：先说问题，再给代码。

   代码如下：
   {{code}}
   ```

   这里新增了两个变量 `sourceLang` 和 `targetLang`，系统会自动识别。

3. **保存快照**

4. **切换到自动驾驶舱**，输入 `/refactor`，意图写 `把一段Python爬虫转成Go`，系统推断参数后，一键生成提示词。

---

### 案例三：A/B 对比两个提示词版本

**背景**：你写了一个产品文案提示词，不确定「专业评审风格」和「网感种草风格」哪个效果更好。

**步骤**：

1. **先在「资产管理」创建两个版本**：
   - 版本 A：风格设为「专业评审」
   - 版本 B：风格设为「网感种草」
   
   分别保存。

2. **进入「A/B 竞技场」**

3. **输入测试意图**：`帮我写一段智能手表的推荐文案`

4. **左侧选择版本 A**，**右侧选择版本 B**

5. **点击「开始对比测试」**

6. **观察两侧输出**，根据内容质量决定哪个版本更合适

7. **点击获胜方的「晋级为主力版本」**

## 5. 提示词编写技巧

### 5.1 变量命名规范

模板中用 `{{变量名}}` 定义参数：

```
✅ 好的写法：
请为产品「{{product}}」写一篇面向{{audience}}的推广文案

❌ 不好的写法：
请为产品「{{产品名称123}}」写一篇面向{{目标人群!!}}的推广文案
```

- 变量名用英文或中文，不含特殊字符
- 每个变量代表一个需要动态填充的元素

### 5.2 模板结构建议

一个好的提示词模板通常包含这四层：

| 层 | 示例 | 说明 |
|----|------|------|
| **角色定位** | 「你是一个资深小红书写手」 | 告诉 AI 它是谁 |
| **输入变量** | `写主题：{{topic}}` | 由用户提供的内容 |
| **格式要求** | 「控制在300字，使用Emoji」| 输出规范 |
| **风格约束** | `风格：{{tone}}` | 语气和表达方式 |

### 5.3 让 AI 更容易理解你的意图

在自动驾驶舱输入意图时，尽量具体：

```
✅ 「帮我写一篇杭州独立咖啡馆的探店文案，温暖治愈风格」
❌ 「咖啡」
```

信息量越大，系统推断的参数越精准。

## 6. 常见问题

### Q：我的数据安全吗？

**安全。** PromptX 是一个纯前端应用，所有数据保存在你的浏览器中：

- 提示词模板存在浏览器的 `IndexedDB` 中
- API Key 存在 `localStorage` 中
- 没有后端服务器，不会上传任何数据
- GitHub Pages 只托管静态 HTML/CSS/JS 文件

### Q：换了电脑/浏览器，数据会丢吗？

**当前版本会丢。** 因为数据存在浏览器本地，不会同步。计划中的功能包括导出/导入。

临时方案：在「资产管理」中手动备份模板文本到记事本。

### Q：支持的 AI 服务商不够怎么办？

选「自定义」模式，填入任意兼容 OpenAI `/v1/chat/completions` 协议的 API 地址。适用于：

- Ollama 本地部署（`http://localhost:11434/v1`）
- One API 聚合网关
- 任何第三方代理服务

### Q：没有 API Key 能用吗？

可以。即使不配置 API Key，模板管理和提示词生成功能也能正常使用（只是最终的 AI 调用会在实际 API 接入时生效）。你可以先用 PromptX 管理模板、生成提示词文本，然后手动复制到 ChatGPT 等工具。

### Q：为什么我的提示词变量没有被识别？

确保模板中使用的是双花括号 `{{变量}}` 格式，且变量名不含空格或特殊字符。

### Q：浏览器扩展和 Web 应用有什么区别？

| | Web 应用 | 浏览器扩展 |
|--|---------|-----------|
| 使用场景 | 管理模板、对比效果 | 在任意网页输入框里唤醒 |
| 安装 | 打开网址即用 | 需从 Chrome Web Store 安装 |
| 唤起方式 | 点击左侧导航 | 在输入框按 `\` 键 |

**推荐先用 Web 应用建好模板库，再用扩展快速调用。**
