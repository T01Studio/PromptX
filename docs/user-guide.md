## 1. PromptX 是什么

PromptX（灵犀）是一个**提示词管理引擎**。

你在日常使用 ChatGPT、DeepSeek、Kimi 等 AI 工具时，是否遇到过这些痛点：

- 每次聊天都要从笔记软件里复制粘贴相同的提示词开头
- 提示词里的 `{{变量}}` 要手动查找替换，一不留神就删错
- 同一个任务，不同提示词的效果好坏全靠感觉，没法系统对比

PromptX 的解决思路很简单：**你管理提示词模板，AI 帮你填参数**。

架构上，它是一套工具组合：

| 组件 | 说明 | 对象 |
|------|------|------|
| **Web 应用** | GitHub Pages 托管的前端，核心操作界面 | 所有人 |
| **浏览器扩展** | 安装在 Chrome 中，在任何网页输入框里按 `\` 就能唤起提示词 | 深度用户 |

> **本文档涵盖 Web 应用和浏览器扩展的全部用法。** 浏览器扩展详见 [第 5 节](#5-浏览器扩展安装与使用)。

## 2. 快速上手指南（5 分钟）

### 2.1 打开应用

访问：**[https://t01studio.github.io/PromptX/](https://t01studio.github.io/PromptX/)**

你会看到一个深色主题界面，左侧有四个导航入口：

```
PromptX
─────────────────
🚗 自动驾驶舱      ← 主操作区
📦 资产管理        ← 管理提示词库
⚔️  A/B 竞技场     ← 对比测试
⚙️  设置中心        ← API、备份、重置
📖 使用文档        ← 本文档（左下角）
🧩 浏览器扩展      ← 插件安装指南（左下角）
```

### 2.2 三步跑通第一个提示词

**第一步：进入「自动驾驶舱」**

点击左侧「自动驾驶舱」，看到对话界面。

**第二步：输入 `/xhs` 然后按 Enter**

在底部输入框输入 `/xhs`，会弹出一个悬浮菜单，显示「小红书种草文案」。按 `Enter` 或点击它。

**第三步：输入意图并发送**

输入框下方会显示已选择的模板名。现在输入你的意图，比如：

```
帮我写一篇杭州小众咖啡馆探店的文案
```

按 `Enter` 发送。系统会自动识别参数（主题、风格、字数）并展示确认面板。

**第四步：确认参数并生成**

检查面板中各参数的值，如需修改点击「微调参数」，确认后点击「直接生成」。

最终提示词会显示在对话区域，鼠标悬停时右上角出现复制按钮，直接复制到 ChatGPT 等工具使用即可。

> **五分钟你学会了核心流程：选模板 → 说意图 → 确认参数 → 得到提示词。**

## 3. 四大模块详解

### 3.1 自动驾驶舱

这里是 PromptX 的核心交互区：**在底部输入框里用 `/` 唤起模板，用自然语言描述意图，系统自动匹配参数并生成最终提示词。**

整个流程分五个步骤，对应五种界面状态：

```
┌─────────┐    ┌──────────┐    ┌─────────────┐    ┌──────────┐    ┌──────────┐
│  input  │ → │ 选择模板  │ → │intent-confirm│ → │generating│ → │  result  │
│  待机   │    │(Ghost UI)│    │  确认参数    │    │ 生成中…   │    │ 可复制   │
└─────────┘    └──────────┘    └─────────────┘    └──────────┘    └──────────┘
```

#### 第一步：唤起模板（Slash Command）

在底部输入框中输入 `/`，立刻弹出一个悬浮菜单（Ghost UI），列出所有可用模板。继续输入字母可以实时过滤：

| 输入 | 效果 |
|------|------|
| `/` | 显示全部模板 |
| `/x` | 只显示命令或标题含 `x` 的（如「小红书种草文案」） |
| `/xhs` | 精确匹配 `xhs` 命令 |
| `/refactor` | 匹配「代码重构专家」 |
| `/repo` | 模糊匹配「工作周报生成器」 |

**键盘操作**：

| 按键 | 作用 |
|------|------|
| `↑` `↓` | 在模板列表中上下移动高亮 |
| `Enter` 或 `Tab` | 选中当前高亮的模板 |
| `Escape` 或 `空格` | 关闭 Ghost UI 菜单 |

> 提示：鼠标悬停在模板上也会自动高亮，点击即可选中。

#### 第二步：选中模板后

模板被选中后：
- Ghost UI 菜单消失
- `/xhs` 这段文字从输入框中**自动删除**（不会出现在最终文本中）
- 输入框左侧出现一个**蓝色标签**，显示已选模板名称（如 `⌘ 小红书种草文案`）
- 输入框 placeh​​older 变为 `请用一句话描述意图，AI将自动补全 3 个参数...`

此时你可以用自然语言描述你想做什么，例如：

```
帮我写一篇杭州小众咖啡馆探店的文案
```

不需要写完整的提示词，只需要一句简单的意图描述。

> 如果要**取消选中**的模板：在输入框为空时按 `Backspace` 即可。

#### 第三步：确认参数（intent-confirm 面板）

输入意图后按 `Enter`，输入框上方会弹出一个**参数确认面板**。系统会根据你输入的关键词自动推演参数值：

> 当前参数推演规则（代码真实逻辑）：
> - 输入中含「咖啡」→ `topic = "咖啡探店"`
> - 输入中含「探店」→ `topic = "周末探店"`
> - 以上都不含 → `topic = "日常分享"`
> - `tone` 固定为 `"真实体验/情绪饱满"`
> - `length` 固定为 `"300"`

面板中会列出模板的每个变量及推演出的值，例如：

```
topic        咖啡探店
tone         真实体验/情绪饱满
length       300
```

**修改参数的方式**：

| 操作 | 按钮 | 效果 |
|------|------|------|
| 接受默认值 | 点击「直接生成」 | 用当前参数生成最终提示词 |
| 手动修改参数 | 点击「微调参数」 | 每个参数变成可编辑输入框，修改后点「完成微调」返回 |

> 微调模式下，点击参数值的文本框直接输入新值即可（如把 `300` 改成 `500`，或把 `咖啡探店` 改成 `美妆测评`）。

#### 第四步：生成中

点击「直接生成」后，输入框上方出现金色提示条：「🔄 正在为您生成...」（持续约 1.5 秒的过渡动画）。

#### 第五步：查看结果并复制

生成完成后，最终提示词以对话气泡形式展示在聊天区域：

- **你的消息**：`[小红书种草文案] 帮我写一篇杭州小众咖啡馆探店的文案`（右侧金色气泡）
- **系统回复**：`✅ 已为您生成最终提示词，可直接复制使用：\n\n你是一个小红书爆款文案专家。请为主题：咖啡探店 撰写一篇种草文案...`（左侧灰色气泡）

鼠标**悬停**在系统回复气泡上时，右上角出现 📋 **复制按钮**，点击即可一键复制提示词文本。复制后可直接粘贴到 ChatGPT、Claude 等 AI 工具中。

> 生成完成后，输入框自动恢复为初始状态（模板取消选中、文本清空），可以继续下一个操作。

#### 完整操作速查卡

```
输入 /xhs → 看到菜单 → 按 Enter 选中
输入意图 "杭州咖啡探店文案"
按 Enter → 看到参数面板 → 可点「微调参数」改值
点「直接生成」→ 等待生成 → 悬停气泡 → 点复制 → 粘贴到 ChatGPT
```

### 3.2 资产管理

这里是你的「提示词图书馆」。管理所有模板的增删改查和版本历史，数据**自动保存到浏览器本地**，刷新页面不会丢失。

#### 界面布局（三栏）

```
┌──────────────┬──────────────────────────┬──────────────┐
│  提示词列表    │      模板编辑器            │  版本追踪     │
│  (320px)     │       (自适应)            │  (320px)     │
│              │                          │  可收起 ▸     │
│  📋 xhs      │  ┌────────────────────┐  │  ● 当前版本   │
│  📋 refactor │  │ {{topic}} ...       │  │  ○ 历史版本1  │
│  📋 report   │  │  ...               │  │  ○ 历史版本2  │
│              │  └────────────────────┘  │              │
└──────────────┴──────────────────────────┴──────────────┘
```

#### 浏览和筛选提示词

左侧面板列出所有模板，每条显示：
- 名称（如「小红书种草文案」）
- 命令（如 `/xhs`）
- 风格（如「真实体验/种草」）

**搜索**：顶部搜索框输入关键词，**按名称和命令**同时搜索。例如输入「代码」会匹配「代码重构专家」（名称含代码）和 `/sql`（无需匹配，因为 SQL 属于代码重构分类）。

**分类筛选**：点击「全部」「自媒体爆款」「代码重构」「公文写作」「艺术绘画」切换视图。

#### 编辑提示词模板

点击左侧任意提示词，中间区域显示编辑器：

1. **头部信息区**：模板名称、`/命令`、参数数量（如「自动参数: 3」）、执行风格
2. **适用场景标签**：如「探店」「美妆」「生活方式」
3. **底层模板编辑器**：一个大文本框，可直接编辑提示词文本

模板中的 `{{变量名}}` 会被**金色高亮**显示。编辑器上方有一个**不可编辑的预览层**，用于实时显示变量高亮效果。

#### 修改模板的两种操作

| 操作 | 触发方式 | 效果 |
|------|---------|------|
| **实时编辑** | 在文本框中直接打字 | 修改立即可见，但**尚未创建版本快照** |
| **保存快照** | 点击右上角「保存快照」 | 当前内容写入版本历史，右侧时间线新增一条记录 |

> 未保存的修改在版本追踪面板中显示为「当前未命名版本 · 修改未保存...」（金色圆点）。

#### 版本管理（完整工作流）

版本追踪面板在编辑器右侧，以**时间线**形式展示所有历史快照：

```
● 当前未命名版本        ← 金色圆点，修改未保存
│
○ 历史版本 2           ← 白色圆点，已保存
│  2026/6/15 10:30
│  [回滚至此版本]
│
○ 历史版本 1
│  2026/6/14 18:00
│  [回滚至此版本]
```

**典型版本管理流程**：

```
1. 选中「小红书种草文案」模板
2. 编辑模板文本（比如增加一个 {{platform}} 变量）
3. 点击「保存快照」→ 版本历史新增一条记录
4. 继续修改 → 如果改坏了，点击某个历史版本的「回滚至此版本」
5. 模板文本立即恢复为选中的历史版本内容
6. 如果确定这是最终版 → 再次「保存快照」固化
```

**对每个历史版本你可以**：
- 查看：版本编号（倒序，版本 1 最早）+ 保存时间戳
- 回滚：点击「回滚至此版本」，编辑器中的模板文本立刻恢复为该版本的完整内容
- 版本追踪面板可收起：点击标题栏右侧 **→ 按钮**，腾出更多编辑空间

#### 删除提示词

选中一个提示词后，右上角「保存快照」旁边有「**删除**」按钮（红色文字）。点击后弹出确认对话框，确认后立即从列表中移除。**此操作不可撤销**，建议先导出备份（见 [设置中心](#34-设置中心)）。

#### 新增提示词模板（两种方式）

点击左上角 **➕** 按钮，弹出「新增系统能力」弹窗，有两种模式：

**方式一：对话提纯（推荐）**

从 AI 聊天记录中自动提取提示词结构：

1. 选择「对话提纯」标签（默认选中）
2. 把你和 AI 的对话历史粘贴到文本框
3. 点击「一键提取」→ 系统模拟分析（约 2.5 秒加载动画）
4. 提取结果自动展示：能力名称、唤起命令、自动变量列表、结构化底层模板
5. 确认后点击「保存至资产库」

**对话提纯示例输出**：
```
能力名称: 小红书探店方法论
唤起命令: /xhs_pro
变量: 亮点、感受细节、字数
模板: 定位：我是一个小红书爆款探店写手...

写作结构：
1. 【吸睛标题】：...
2. 【核心卖点】：深入描写 {{亮点}}
3. 【体验感受】：结合个人视角谈 {{感受细节}}

要求：字数控制在 {{字数}} 字左右，多使用 Emoji。
```

> 提取出的变量会自动高亮，模板可直接在编辑器中继续修改。

**方式二：手动创建**

传统表单输入模式（当前版本为演示占位，推荐使用对话提纯）。

### 3.3 A/B 竞技场

用**同一个测试意图**，同时运行两个不同的提示词模板，左右分屏对比生成质量，决定哪个版本是你真正想要的。

#### 界面布局

```
┌─────────────────────────────────────────────────────┐
│  测试意图输入框   [                                                   ] [开始对比测试] │
├──────────────────────────┬──┬────────────────────────┤
│   模型方案 A              │↔│   模型方案 B              │
│   [下拉选择模板]          │  │   [下拉选择模板]          │
│  ┌────────────────────┐  │  │  ┌────────────────────┐  │
│  │                    │  │  │  │                    │  │
│  │  左侧输出区域       │  │  │  │  右侧输出区域       │  │
│  │  (流式打印)        │  │  │  │  (流式打印)        │  │
│  │                    │  │  │  │                    │  │
│  └────────────────────┘  │  │  └────────────────────┘  │
│       [🏆 晋级为主力版本]  │  │       [🏆 晋级为主力版本]  │
└──────────────────────────┴──┴────────────────────────┘
```

#### 完整操作步骤

**第一步：输入测试意图**

在顶部输入框写一个**勾子性问题**，两个模板会用同样的输入去跑：

```
帮我写一段智能手表的推荐文案
```

> 确保两个方案拿到的是完全相同的输入，这样对比才公平。

**第二步：选择两个要对比的方案**

- 左侧下拉框：选择方案 A（如「小红书种草文案」）
- 右侧下拉框：选择方案 B（如「代码重构专家」或其他自定义模板）

下拉框中列出你库中所有提示词模板。可以选两个不同的模板，也可以选同一个模板的不同版本。

**第三步：点击「开始对比测试」**

按钮变为禁用状态并显示「生成中...」，两侧同时开始**流式输出**结果。

> 当前版本是模拟流式打印：字符逐个出现（约每 20ms 一个字符），模拟真实 AI API 的流式效果。

**第三步（等）：观察流式输出**

两侧输出区域同时打字，可以实时看到差异：

```
方案 A 输出示例：
【结果对比 A】
这是一份极具专业视角的文案：
关于您的要求：帮我写一段智能手表的推荐文案，
我们为您提供如下建议...
[由系统引擎自动生成]

方案 B 输出示例：
【结果对比 B】
🔥爆款预警！家人们谁懂啊，今天实测了：
帮我写一段智能手表的推荐文案，真的绝绝子！
[由系统引擎自动生成]
```

**第四步：对比并决定**

加载完成后，两侧底部各出现一个「🏆 **晋级为主力版本**」按钮。根据输出质量决定哪个方案更合适：

| 判断维度 | 看什么 |
|----------|--------|
| 风格匹配度 | 方案 A 偏专业严谨，方案 B 偏网感/种草？你的内容适合哪类？ |
| 变量填充质量 | 哪个方案自动填充的参数更精准？ |
| 输出结构 | 哪个方案的输出格式更清晰、更易于直接使用？ |

**应用场景**

| 场景 | 操作 |
|------|------|
| **提示词迭代** | 方案 A = 旧版模板，方案 B = 改过的新版，看哪个更好 |
| **风格对比** | 方案 A = 专业风，方案 B = 网感风，决定你的内容调性 |
| **AB 后固化** | 确定获胜方 → 把输的一方模板改掉或删除 → 保存获胜方的快照 |
| **多轮 PK** | 改一下测试意图再跑一次，验证模板的稳定性 |

> 当前版本的生成是模拟数据，不是真实调用 AI API。主要帮助你**比较模板结构、参数设计和输出风格**，而非实际模型能力。接入 API 后可替换为真实调用。

### 3.4 设置中心

设置中心整合了三个功能：**API 配置**、**数据导入导出**、**重置数据**。

#### 3.4.1 API 配置

配置你连接 AI 大模型的方式。所有配置仅保存在你的浏览器中，**不会上传到任何服务器**。

**支持的 AI 服务商**

| 服务商 | 默认接口地址 | 可用模型 |
|--------|-------------|----------|
| **Google Gemini** | `generativelanguage.googleapis.com` | gemini-2.5-flash、gemini-2.5-pro、gemini-2.0-flash |
| **DeepSeek** | `api.deepseek.com` | deepseek-chat、deepseek-reasoner |
| **OpenAI** | `api.openai.com` | gpt-4o、gpt-4o-mini、gpt-4-turbo |
| **自定义** | 随意填入 | 随意填入（兼容 OpenAI 协议） |

**配置步骤**

1. 点击左侧「设置中心」
2. 选择你要用的服务商
3. 填入 API Key（从对应服务商后台获取）
4. 选择或输入模型名称
5. 点击「保存配置」

**如何获取 API Key**

| 服务商 | 获取地址 | 说明 |
|--------|---------|------|
| Gemini | [aistudio.google.com/apikey](https://aistudio.google.com/apikey) | 免费额度充足|
| DeepSeek | [platform.deepseek.com/api_keys](https://platform.deepseek.com/api_keys) | 性价比高 |
| OpenAI | [platform.openai.com/api-keys](https://platform.openai.com/api-keys) | 需充值 |

> 自定义接口支持任何兼容 OpenAI `/v1/chat/completions` 协议的服务（如 Ollama 本地部署、One API 聚合网关等）。

#### 3.4.2 数据备份（导入 / 导出）

PromptX 的数据存储在浏览器本地，**换电脑或清除浏览器缓存后数据会丢失**。使用导入/导出功能可以解决这个问题。

**导出备份（公司电脑 → 文件）**

1. 进入「设置中心」
2. 找到「数据备份」区域
3. 点击「**导出备份**」
4. 浏览器自动下载一个 JSON 文件，命名如 `promptx-backup-2026-06-15.json`

导出的文件包含：

| 内容 | 说明 |
|------|------|
| 所有提示词模板 | 模板内容、变量、版本历史 |
| 所有分类 | 如「自媒体爆款」「代码重构」等 |
| API 配置 | 提供商、API Key、模型等 |

**导入恢复（文件 → 家电脑）**

1. 把备份文件传到新电脑（U 盘 / 微信 / 飞书 / 邮件）
2. 打开 PromptX，进入「设置中心」
3. 点击「**导入备份**」，选择 JSON 文件
4. 系统自动验证格式，导入成功后立即生效

> 导入时会做格式校验。非法 JSON 或缺失关键字段会被拒绝，并提示具体错误信息。

**典型工作流**

```
周五下班前：公司电脑 → 导出备份 → 文件发到微信「文件传输助手」
周六在家：  家用电脑 → PromptX → 导入备份 → 继续工作
周一上班：  公司电脑 → 如果在家改了模板，再次导出 → 回到公司导入同步
```

**安全提醒**

- 备份文件**包含 API Key（明文）**，传输时建议用加密方式
- 不要把备份文件上传到公开网盘
- 使用完毕后可以从微信聊天记录中删除

#### 3.4.3 重置所有数据

在「设置中心」底部找到红色区域「**重置所有数据**」，点击后清除：
- 所有提示词模板
- 所有分类
- API 配置

> 数据完全清空，恢复为初始状态。**建议先导出备份！**

### 案例一：从零创建小红书文案模板

**背景**：你经常需要为不同品牌写小红书种草文案，但每次都要重新构思结构。

**步骤**：

1. **打开 PromptX** → 点击左侧「资产管理」

2. **点击 ➕ 新增** → 选择「对话提纯」

3. **粘贴一段你满意的历史对话**：
   ```
   我：帮我写一篇小红书种草文案
   AI：好的，什么产品？
   我：国货精华液
   AI：需要突出什么卖点？
   我：成分安全、敏感肌可用
   AI：风格偏好？
   我：真实测评风格
   我：字数300左右
   AI：[生成了文案...]
   ```

4. **点击「一键提取」**，系统分析后输出：
   - 能力名称：国货护肤品测评
   - 唤起命令：/skincare
   - 变量：产品、卖点、风格、字数
   - 模板：（结构化提示词）

5. **确认无误后点击「保存至资产库」**

6. **切换到「自动驾驶舱」**，输入 `/skincare`，输入意图 `推荐一款平价眼霜`，系统自动填充参数 → 生成！

> 这就是**从经验到模板**的完整流程。一次提纯，永久复用。

### 案例二：打造代码重构专家

## 5. 浏览器扩展安装与使用

PromptX 浏览器扩展让你**在任意网页输入框中按 `\` 键**即可唤起提示词模板，不再需要离开当前页面复制粘贴。

### 5.1 核心优势

| 对比 | 不用扩展 | 用扩展 |
|------|---------|--------|
| 切换 ChatGPT 写文案 | 打开 Web 应用 → 生成提示词 → 复制 → 切回 ChatGPT → 粘贴 | 在 ChatGPT 输入框直接按 `\xhs` → 选中模板 → 提示词自动填入 |
| 在 Claude 重构代码 | 切回 IDE 找模板 → 复制 → 回 Claude 粘贴 | 在 Claude 输入框按 `\refactor` → 一键注入 |
| 任何网页输入框 | 需手动打字 | 按 `\` 即刻唤起菜单 |

### 5.2 安装方式（二选一）

#### 方式一：Chrome Web Store（推荐，待上架）

> 扩展正在审核中，上架后搜索「PromptX」一键安装即可。

#### 方式二：开发者模式手动加载（当前可用）

1. 下载扩展文件夹  
   从 [GitHub 仓库 `extension/` 目录](https://github.com/T01Studio/PromptX/tree/master/extension) 下载整个 `extension` 文件夹到本地

2. 打开 Chrome 扩展管理页  
   地址栏输入 `chrome://extensions/`，回车

3. 开启「**开发者模式**」  
   页面右上角的开关，点一下变蓝色

4. 点击「**加载已解压的扩展程序**」  
   选择你刚下载的 `extension` 文件夹

5. 看到 PromptX 图标出现在浏览器工具栏右侧，安装完成

### 5.3 快速体验（30 秒）

### 5.7 技术原理（开发者向）

扩展采用 Manifest V3 架构，注入机制自动适配两种输入框：

- **标准输入框**（`<textarea>` / `<input>`）：使用 `nativeInputValueSetter` 设值并派发 `input` + `change` 合成事件，确保 React / Vue 框架能正确感知
- **富文本输入框**（`contenteditable`）：选区恢复 + DOM 文本节点替换，降级时自动复制到剪贴板

### 5.8 自定义提示词

编辑 `extension/prompts.js` 文件，按同样格式添加你的指令：

```js
{
  id: 'p-99',
  title: '你的模板名称',
  command: 'mycmd',        // 唤起指令
  category: '自定义',
  template: `你的提示词文本，支持 {{变量}} 哦...`,
  variables: ['变量'],
},
```

保存后，在 `chrome://extensions/` 点击 PromptX 卡片的刷新按钮即可生效。

### 5.9 常见问题

**Q：按 `\` 没反应？**  
A：确认扩展已安装并启用（浏览器右上角有 PromptX 图标）。检查 `chrome://extensions/` 中扩展状态。

**Q：提示词注入后 ChatGPT 无法发送？**  
A：注入引擎已适配 React 合成事件。如果仍有问题，稍等 0.5 秒让框架消化输入事件，或手动在文本末尾加一个空格再删掉即可触发 React 状态更新。

**Q：能和 Web 应用联动吗？**  
A：当前版本扩展内置了独立提示词库（`prompts.js`）。后续版本计划支持扩展读取 Web 应用中存储的模板，实现无缝同步。

## 6. 提示词编写技巧

### 6.1 变量命名规范

模板中用 `{{变量名}}` 定义参数：

```
✅ 好的写法：
请为产品「{{product}}」写一篇面向{{audience}}的推广文案

❌ 不好的写法：
请为产品「{{产品名称123}}」写一篇面向{{目标人群!!}}的推广文案
```

- 变量名用英文或中文，不含特殊字符
- 每个变量代表一个需要动态填充的元素

### 6.2 模板结构建议

一个好的提示词模板通常包含这四层：

| 层 | 示例 | 说明 |
|----|------|------|
| **角色定位** | 「你是一个资深小红书写手」 | 告诉 AI 它是谁 |
| **输入变量** | `写主题：{{topic}}` | 由用户提供的内容 |
| **格式要求** | 「控制在300字，使用Emoji」| 输出规范 |
| **风格约束** | `风格：{{tone}}` | 语气和表达方式 |

### 6.3 让 AI 更容易理解你的意图

在自动驾驶舱输入意图时，尽量具体：

```
✅ 「帮我写一篇杭州独立咖啡馆的探店文案，温暖治愈风格」
❌ 「咖啡」
```

信息量越大，系统推断的参数越精准。

## 7. 常见问题

### Q：我的数据安全吗？

**安全。** PromptX 是一个纯前端应用，所有数据保存在你的浏览器中：

- 提示词模板存在浏览器的 `localStorage` 中
- API Key 存在 `localStorage` 中
- 没有后端服务器，不会上传任何数据
- GitHub Pages 只托管静态 HTML/CSS/JS 文件
- 其他用户访问同一个网址，看到的是他们自己的浏览器数据，看不到你的

### Q：换了电脑/浏览器，数据怎么转移？

**使用导入/导出功能。** 进入「设置中心」→「数据备份」：

1. 旧电脑上点击「**导出备份**」，得到一个 JSON 文件
2. 把文件传到新电脑（微信 / U 盘 / 飞书 / 邮件均可）
3. 新电脑上打开 PromptX → 设置中心 → 点击「**导入备份**」→ 选择文件
4. 提示词、分类、API 配置全部恢复

> 详细说明见 [设置中心 - 数据备份](#342-数据备份导入--导出)。

### Q：支持的 AI 服务商不够怎么办？

选「自定义」模式，填入任意兼容 OpenAI `/v1/chat/completions` 协议的 API 地址。适用于：

- Ollama 本地部署（`http://localhost:11434/v1`）
- One API 聚合网关
- 任何第三方代理服务

### Q：没有 API Key 能用吗？

可以。即使不配置 API Key，模板管理和提示词生成功能也能正常使用（只是最终的 AI 调用会在实际 API 接入时生效）。你可以先用 PromptX 管理模板、生成提示词文本，然后手动复制到 ChatGPT 等工具。

### Q：为什么我的提示词变量没有被识别？

确保模板中使用的是双花括号 `{{变量}}` 格式，且变量名不含空格或特殊字符。
